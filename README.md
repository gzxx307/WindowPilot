# WindowPilot

针对 Windows 操作系统的自动化工具，旨在帮助用户更高效地完成日常任务和工作流程。尤其在窗口数量过多的情况下，WindowPilot 可以帮助用户快速切换、管理和操作窗口，提高工作效率。

## 功能

- **接管窗口**：拖动任意窗口到左侧侧边栏，WindowPilot 自动接管并布局
- **多种布局**：支持堆叠抽屉 / 四分区 / 左右分屏 / 上下分屏，一键切换
- **抽屉切换**：堆叠模式下点击侧边栏条目即可切换，告别 Alt+Tab 大海捞针
- **全局热键**：支持在窗口失去焦点时仍能够操作窗口

以下是默认热键列表：

| 热键            | 功能       |
|---------------|----------|
| `Ctrl+Alt+G`  | 抓取当前活动窗口 |
| `Ctrl+Alt+←`  | 切换到上一个窗口 |
| `Ctrl+Alt+→`  | 切换到下一个窗口 |
| `Ctrl+Alt+Q`  | 切换到四分区布局 |

**小巧思：**

- 由于Windows权限管理问题，当程序以管理员身份运行时则无法接管非管理员身份运行的窗口，反之类似。于是我设置程序会自动跳过管理员权限窗口、系统窗口、最小尺寸过大的窗口，不会乱接管东西
- 把窗口从侧边栏释放后，会回到被接管前的位置和大小
- 接管窗口后会自动调整窗口样式，去掉标题栏的最小化和全屏按钮，避免窗口被误操作拖动或调整大小

## 开发前准备

**个人开发环境**

- Windows 11
- .Net 8 SDK
- Visual Studio 2026 / Rider

**理论环境要求**

- Windows 10 / 11
- [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
- Visual Studio 2022+ 或 Rider（随便用哪个，能跑就行）

**克隆与运行**

```bash
git clone https://github.com/gzxx307/WindowPilot.git
cd WindowPilot

# 直接运行
dotnet run

# 构建
dotnet build

# 发布为独立可执行文件
dotnet publish -c Release -r win-x64 --self-contained
```

如果你想在 Visual Studio 或 Rider 中运行，直接打开项目文件（.sln 或 .csproj）即可。

## 关于未来

由于WPF本身的性能问题，WindowPilot在窗口数量较多时可能会出现卡顿和响应迟钝的情况，尤其是在堆叠抽屉模式下。未来计划优化性能，可能会考虑使用更高效的渲染方式或者引入其他技术来提升用户体验（例如 electron / tauri 等前端技术，这可能会导致项目级重构）。
